# AGENTS.md

## 项目名称

湖北省内普通高考专科批志愿推荐网站

## 项目定位

本项目是一个面向湖北普通高考高职高专普通批考生和家长的志愿填报辅助工具。

核心目标是帮助用户根据首选科目、分数、位次、院校性质、城市和专业方向，快速筛选湖北省内专科院校专业组，并生成冲、稳、保、兜底志愿草表。

本项目不是官方志愿填报系统，不接入官方填报平台，不自动提交志愿，只做公开资料整理、志愿参考和草表生成。

## 目标用户

1. 湖北普通高考首选物理考生
2. 湖北普通高考首选历史考生
3. 200–400 分段专科批考生
4. 没过本科线或本科滑档风险考生
5. 想读湖北省内专科的考生
6. 想优先选择公办专科的考生
7. 想了解专本联合培养的考生
8. 想报护理、计算机、铁路、电力、财经、学前教育等热门专业的考生

## 技术栈

请使用以下技术栈：

* Vue 3
* Vite
* Element Plus
* Tailwind CSS
* SheetJS
* localStorage
* CSV / JSON 静态数据

第一版不做后端、不做登录、不做支付、不做数据库。

## 核心功能

1. 首页展示产品说明和查询入口。
2. 用户可以输入首选科目、分数、位次。
3. 用户可以筛选院校性质：公办、民办、不限。
4. 用户可以筛选城市：武汉、省内其他、不限。
5. 用户可以筛选专业方向：计算机、护理、铁路、电力、财经、学前教育、建筑、机械、旅游等。
6. 用户可以选择是否只看专本联合培养。
7. 系统根据 2026 招生计划和 2025、2024、2023 投档线生成推荐。
8. 推荐结果分为冲一冲、稳一稳、保一保、兜底。
9. 用户可以把院校专业组加入志愿清单。
10. 志愿清单最多 30 个院校专业组。
11. 志愿清单支持删除、清空、调整顺序。
12. 志愿清单保存到 localStorage。
13. 支持导出 Excel 志愿草表。
14. 所有页面必须展示免责声明。

## 推荐算法原则

1. 优先使用位次，不要只看分数。
2. 只推荐 2026 年招生计划中仍然存在的湖北省内院校专业组。
3. 院校专业组代码以 2026 招生计划为准。
4. 2025、2024、2023 投档线只作为历史参考。
5. 用户选择公办时，只展示公办院校。
6. 用户选择专业方向时，优先匹配专业名称、专业方向和备注字段。
7. 如果用户没有输入位次，可以使用 2026 一分一段表把分数换算成位次。
8. 没有历史投档线的数据必须标注“历史数据不足”。

推荐分类规则：

* 冲一冲：学校去年最低位次比用户位次靠前 5%–20%
* 稳一稳：学校去年最低位次与用户位次前后 5% 以内
* 保一保：学校去年最低位次比用户位次靠后 5%–25%
* 兜底：学校去年最低位次比用户位次靠后 25% 以上

## 风险标签

以下情况要显示风险标签：

1. 民办院校
2. 学费较高
3. 中外合作办学
4. 校企合作
5. 招生计划较少
6. 备注中有身体条件限制
7. 备注中有单科成绩限制
8. 备注中有性别限制
9. 2026 招生计划较上一年明显减少
10. 缺少近三年投档线数据

## UI 设计要求

整体风格：

* 干净
* 现代
* 可信任
* 适合学生和家长
* 不要花哨
* 不要像后台系统
* 不要过度堆叠信息

布局要求：

* 页面最大宽度建议 1200px
* 首页要有清晰 Hero 区域
* 查询表单要突出
* 推荐结果用卡片和表格结合
* 移动端表格可转换为卡片
* 按钮要明显
* 页面留白要充足
* 字体大小要适合家长阅读
* 重要风险提示要醒目但不要夸张

颜色建议：

* 主色：蓝色或蓝紫色
* 背景：浅灰白
* 卡片：白色
* 冲一冲：橙色标签
* 稳一稳：蓝色标签
* 保一保：绿色标签
* 兜底：灰色或深绿色标签
* 风险提示：红色或橙色

## 合规要求

禁止实现以下功能：

1. 自动登录官方志愿填报系统
2. 自动提交志愿
3. 收集身份证号
4. 收集准考证号
5. 收集官方填报系统密码
6. 承诺包录取
7. 承诺一定不滑档
8. 把推荐结果说成官方结果

所有页面必须展示以下免责声明：

本工具仅供湖北普通高考高职高专普通批志愿填报参考，推荐结果基于公开招生计划、历年投档线和一分一段数据测算，不代表官方录取结果。最终填报请以湖北省招生数智综合平台、《湖北招生考试》杂志、湖北省教育厅、省招办及高校招生章程为准。

## 部署要求

第一版必须支持静态部署。

要求：

1. npm install 可以正常安装依赖。
2. npm run dev 可以启动本地开发环境。
3. npm run build 可以成功打包。
4. 打包后生成 dist 目录。
5. dist 目录可以部署到 Cloudflare Pages、Vercel、Netlify 或 GitHub Pages。
6. 不要依赖 Node 后端运行时。
7. 路由建议使用 hash 模式，避免静态部署刷新 404。
